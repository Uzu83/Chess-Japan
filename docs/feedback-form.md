# フィードバック用 Google フォーム 仕様 & Gemini プロンプト

アプリ内「フィードバック」ボタンから開く Google フォームを作るためのドキュメント。
作成後、フォームの公開 URL を環境変数 `VITE_FEEDBACK_URL` に設定するとアプリに導線が出る。

## 作り方（2通り）

### A. Gemini で自動生成（推奨・速い）

Google フォームの編集画面右上にある **「Gemini に手伝ってもらう / Help me create」** を開き、
下の「Gemini プロンプト」をそのまま貼り付ける。生成後、下の「必須/推奨項目」と照合して微調整する。

> 注: Gemini in Google Forms は Workspace 環境・提供地域によって表示されないことがある。
> 出ない場合は B の手動作成を使う。

### B. 手動作成

下の「必須/推奨項目」をそのまま質問として追加していく。

## Gemini プロンプト（コピペ用）

```
チェスと将棋の「1手1手をAIが解説するWebアプリ(Chess-Japan)」のユーザーフィードバック用フォームを日本語で作成してください。回答のハードルを下げるため、必須項目は最小限にし、ほとんどを任意にしてください。次の質問を含めてください:

1. フィードバックの種類（ラジオボタン・必須）: バグ報告 / 機能のリクエスト / 解説の品質について / 使いやすさ(UI/UX) / その他
2. 内容を具体的に教えてください（段落・必須）
3. 解説のわかりやすさの満足度（5段階の均等目盛り・任意。1=わかりにくい〜5=とてもわかりやすい）
4. アプリ全体の満足度（5段階の均等目盛り・任意）
5. どの機能についてですか（チェックボックス・任意）: 棋譜の振り返り / AIと対局しながら解説 / 任意局面の解説 / 解説への質問(対話) / 局面から対局を再開(復習) / その他
6. 使用した端末（ラジオボタン・任意）: スマホ / タブレット / PC
7. 使用したブラウザ（ラジオボタン・任意）: Chrome / Safari / Firefox / Edge / その他
8. （バグの場合）再現手順（段落・任意）
9. 該当する局面のFEN/SFENまたは棋譜があれば貼ってください（段落・任意）
10. スクリーンショットがあれば添付してください（ファイルのアップロード・任意・画像のみ）
11. 返信が必要な場合のみメールアドレス（短文・任意。メール形式を検証）

設定: メールアドレスは自動収集しない。1人につき複数回回答できるようにする。確認メッセージは「フィードバックありがとうございます！いただいた内容は今後の改善に役立てます。」にしてください。
```

## 必須/推奨項目（手動作成・チェック用）

### 必須（2つだけ。回答率を上げるため最小限）

| 項目 | 種類 | 選択肢/備考 |
|---|---|---|
| フィードバックの種類 | ラジオ | バグ報告 / 機能のリクエスト / 解説の品質 / 使いやすさ(UI/UX) / その他 |
| 内容を具体的に | 段落 | 自由記述 |

### 推奨（任意。あると改善に役立つ）

| 項目 | 種類 | 選択肢/備考 |
|---|---|---|
| 解説のわかりやすさ満足度 | 均等目盛り 1–5 | 1=わかりにくい / 5=とてもわかりやすい |
| アプリ全体の満足度 | 均等目盛り 1–5 | — |
| 対象の機能 | チェックボックス | 棋譜振り返り / AI対局解説 / 任意局面 / 対話解説 / 復習(再開) / その他 |
| 端末 | ラジオ | スマホ / タブレット / PC |
| ブラウザ | ラジオ | Chrome / Safari / Firefox / Edge / その他 |
| 再現手順（バグ時） | 段落 | — |
| 該当局面 FEN/SFEN・棋譜 | 段落 | デバッグ再現に有用 |
| スクリーンショット | ファイル添付 | 画像のみ・任意 |
| 連絡先メール | 短文 | 任意・メール形式検証・返信希望時のみ |

### フォーム設定の推奨

- **メールアドレスの自動収集: オフ**（匿名で送れる＝PII最小・心理的ハードル低減。返信が要る人だけ任意で記入）。
- **回答の制限: 複数回可**（ログイン強制しない）。
- **進行状況バー**: オン（項目が増えるため）。
- **確認メッセージ**: 「フィードバックありがとうございます！いただいた内容は今後の改善に役立てます。」
- 回答は Google スプレッドシートにリンクしておくと集計が楽。

---

## アプリ側の連携

- フォーム URL を `.env` の `VITE_FEEDBACK_URL` に設定（例: `VITE_FEEDBACK_URL=https://forms.gle/xxxxx`）。
- 未設定ならボタンは表示されない（`src/App.tsx` で制御済み）。
- 将来、端末/ブラウザ/該当局面を URL パラメータでプリフィルすると回答精度が上がる
  （Google フォームの「事前入力したURLを取得」でフィールド ID を調べて付与）。
